## 准备工作

- 一台 Windows 电脑（运行代理和抓取脚本）
- Windows 桌面微信客户端（版本 3.9.x 或更新）
- 已安装 [uv](https://docs.astral.sh/uv/)（Python 包管理器）

### 安装

```bash
# 进入项目目录
cd wechat

# 安装依赖并创建虚拟环境
uv sync
```

### 验证安装

```bash
uv run wechat-fetcher --help
```

如果看到命令列表（`fetch`、`start-proxy`、`status`、`download`、`run`），说明安装成功。

> **推荐使用 `uv run`**，无需手动激活虚拟环境。如果已激活（`.venv\Scripts\activate`），也可直接用 `python -m wechat_fetcher`。

## 方式A：Windows 桌面微信（推荐）

电脑上已安装微信桌面版的情况下，**无需手机**，全程在同一台电脑上完成。

### A.1 启动代理

打开终端，在项目目录下运行：

```bash
uv run wechat-fetcher start-proxy
```

> 代理会持续运行，**保持此窗口不要关闭**。

### A.2 设置 Windows 系统代理

1. 按 `Win + I` 打开 **设置**
2. 点击左侧 **网络和 Internet** → 右侧 **代理**
3. 在 **手动设置代理** 区域，点击 **使用代理服务器** 旁边的 **设置**
4. 将开关设为 **开**，填写：
   - **代理 IP 地址**：`127.0.0.1`
   - **端口**：`8080`
5. 点击 **保存**

> 操作完成后请**关闭代理设置**。

### A.3 安装 mitmproxy CA 证书

**确保代理仍在运行**，然后：

1. 打开浏览器，访问：`http://mitm.it`
2. 点击 **Windows** 图标下载证书
3. 双击下载的证书文件，打开 **证书导入向导**
4. 存储位置选择 **当前用户** → 下一步
5. 选择 **"将所有证书放入下列存储"** → 点击 **浏览**
6. 选择 **"受信任的根证书颁发机构"** → 确定 → 下一步
7. 点击 **完成**，在弹出的安全警告中点击 **是**

### A.4 捕获公众号参数

1. **完全关闭并重新打开**微信桌面版
2. 登录微信，找到目标公众号
3. 点击右上角头像 → **"全部消息"**，向下滚动加载更多文章
4. 代理终端窗口输出 `Captured params for __biz=...` 即表示成功

此时可**新开另一个终端**执行抓取，代理窗口建议保持运行（便于批量捕获多个公众号）。

### A.5 恢复系统代理

操作完成后**务必关闭系统代理**：
1. 按 `Win + I` → **网络和 Internet** → **代理**
2. 将 **使用代理服务器** 开关设为 **关**

## 方式B：手机微信

### B.1 启动代理

```bash
uv run wechat-fetcher start-proxy --show-ip
```

记下输出的 IP 地址（如 `192.168.1.100`）。

### B.2 手机设置 Wi-Fi 代理

**iOS**：设置 → Wi-Fi → 当前网络旁边的 ⓘ → HTTP 代理 → 手动 → 填入 IP 和端口 `8080`

**Android**：设置 → WLAN → 长按当前 Wi-Fi → 修改网络 → 代理 → 手动 → 填入 IP 和端口 `8080`

### B.3 安装 CA 证书

代理运行时，手机浏览器访问 `http://mitm.it`，按页面指引安装。

### B.4 捕获公众号参数

在手机微信中打开目标公众号，进入"全部消息"，向下滑动加载更多文章。

### B.5 恢复手机网络

操作完成后移除手机代理，建议卸载证书。

## 一键工作流（推荐）

代理捕获参数后，使用 `run` 命令一站式完成 抓取 → 去重 → 下载：

### 对所有已存储账号执行

```bash
uv run wechat-fetcher run --all --days 3
```

### 指定日期范围

```bash
uv run wechat-fetcher run --all --from 2026-03-01 --to 2026-03-31
```

### 指定单个/多个账号

```bash
uv run wechat-fetcher run --biz MzAxxx --biz MzByyy --days 7
```

### 参数过期后自动等待代理

```bash
# 新开终端运行代理
uv run wechat-fetcher start-proxy

# 另一个终端运行 --wait-proxy
uv run wechat-fetcher run --all --days 3 --wait-proxy
```

`--wait-proxy` 模式会轮询等待（最多 5 分钟），检测到参数就绪后自动继续执行。

### 输出

- 文章保存至 `data/articles/<date>_<title>/content.md`
- 已下载文章自动记录到去重索引，下次运行自动跳过
- 命令结束输出摘要：成功 / 跳过 / 失败 数量

## 分步操作

如果需要更精细的控制，可以分步执行。

### 1. 查看已捕获的账号

```bash
uv run wechat-fetcher status
```

输出示例：

```
Stored accounts (data/params):

  MzAwNjQwNzU2NQ==
    Extracted: 2026-05-16T06:38:45+00:00 (120s ago) fresh

去重索引 (data/index/dedup_index.json):
  文章标题: 127  文章URL: 127  账号数: 1
```

### 2. 抓取文章列表

```bash
# 最近 N 天
uv run wechat-fetcher fetch --biz MzAwNjQwNzU2NQ== --days 30

# 指定日期范围
uv run wechat-fetcher fetch --biz MzAwNjQwNzU2NQ== --from 2026-03-01 --to 2026-03-31

# 导出到 JSON 文件
uv run wechat-fetcher fetch --biz MzAwNjQwNzU2NQ== --days 30 --output articles.json
```

`--days` 和 `--from/--to` 互斥，不能同时使用。

### 3. 下载文章为 Markdown

```bash
uv run wechat-fetcher download --input articles.json
```

下载过程自动跳过已存在的文章。输出目录默认为 `data/articles/`。

```bash
# 指定输出目录
uv run wechat-fetcher download --input articles.json --output-dir ./my_articles

# 带 Cookie 下载（用于需要登录的文章）
uv run wechat-fetcher download --input articles.json --cookie "your_cookie_string"
```

## 常用命令参考

| 命令 | 用途 |
|------|------|
| `wechat-fetcher start-proxy` | 启动 MITM 代理，捕获认证参数 |
| `wechat-fetcher start-proxy --show-ip` | 启动代理并显示本机 IP（手机模式） |
| `wechat-fetcher start-proxy --port 9090` | 指定代理端口 |
| `wechat-fetcher status` | 查看已存储账号 + 去重索引统计 |
| `wechat-fetcher status --verbose` | 查看完整参数（含 key/uin） |
| `wechat-fetcher fetch --biz <biz> --days 30` | 抓取最近 30 天文章列表 |
| `wechat-fetcher fetch --biz <biz> --from YYYY-MM-DD --to YYYY-MM-DD` | 按日期范围抓取 |
| `wechat-fetcher fetch --biz <biz> --days 30 --output result.json` | 导出 JSON |
| `wechat-fetcher download --input result.json` | 下载文章并转 Markdown |
| `wechat-fetcher run --all --days 3` | 一键工作流（所有账号） |
| `wechat-fetcher run --biz <biz> --from YYYY-MM-DD --to YYYY-MM-DD` | 一键工作流（指定账号/日期） |
| `wechat-fetcher run --all --days 3 --wait-proxy` | 等待代理就绪后自动运行 |

## 常见问题排查

### A 类问题：Windows 桌面微信

| 问题 | 解决方法 |
|------|---------|
| 设置系统代理后浏览器无法上网 | 确认代理终端窗口仍在运行；操作完成后及时关闭系统代理 |
| 微信桌面版无法登录或网络异常 | 先关闭代理登录微信，登录成功后再开启代理；确认防火墙未拦截 mitmdump |
| 滑动文章列表但没有 "Captured params" 输出 | 确认微信完全退出后重新打开；确认 CA 证书已安装到"受信任的根证书颁发机构"；确保进入"全部消息"页面而非聊天页面 |
| 代理启动报错 mitmdump not found | 运行 `uv sync` 重新安装依赖 |

### B 类问题：手机微信

| 问题 | 解决方法 |
|------|---------|
| 代理启动后手机无法上网 | 确认防火墙允许 8080 端口入站；确认手机和电脑在同一 Wi-Fi |
| mitm.it 页面打不开 | 确认代理仍在运行；尝试 `http://mitm.it`；检查防火墙 |
| 滑动文章列表但没有捕获输出 | iOS 确认证书信任开关已打开；完全关闭微信后重新打开；慢慢滑动让每页加载完成 |
| 某个公众号无法捕获参数 | 进入该公众号后，随机打开一篇文章再返回列表滚动加载 |

### C 类问题：通用

| 问题 | 解决方法 |
|------|---------|
| 拉取时报错 "Key 已过期" | 重新运行 `start-proxy`，在微信中再次浏览该公众号的历史文章，然后重试 |
| 抓取过程中提示 "Rate limited" | 工具会自动指数退避等待，耐心等待即可 |
| 部分文章下载后内容不全 | 微信图文消息可能包含隐藏元素，这是正常现象 |
| 输出结果中多篇文章同一天但 URL 不同 | 正常——微信公众号支持"多图文推送"，一次推送最多 8 篇 |
| 去重索引能否重置 | 删除 `data/index/dedup_index.json` 即可从头开始 |

## 安全提醒

1. **`data/params/` 包含敏感凭证**：存储了你微信账号的接口鉴权参数（`uin` 和 `key`）。拥有这些参数的人可以在有效期内拉取你可见的公众号文章。请勿分享或上传（`data/` 已加入 `.gitignore`）。

2. **CA 证书具有高权限**：mitmproxy 的 CA 证书可以解密你设备上所有 HTTPS 流量。使用完毕后，建议卸载证书：
   - **Windows**：`Win+R` 输入 `certmgr.msc`，展开"受信任的根证书颁发机构"→"证书"，找到 `mitmproxy`，右键 → 删除
   - **iOS**：设置 → 通用 → VPN 与设备管理 → 删除描述文件
   - **Android**：设置 → 安全 → 加密与凭据 → 用户凭据 → 删除 mitmproxy

3. **合规使用**：本工具利用你自身微信账号的合法权限拉取公开文章信息。请合理使用，遵守微信平台的使用条款。
