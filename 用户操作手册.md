## 准备工作

- 一台 Windows 电脑（运行代理和抓取脚本）
- Windows 桌面微信客户端（版本 3.9.x 或更新）
- 已安装 [uv](https://docs.astral.sh/uv/)（Python 包管理器）

安装依赖并初始化虚拟环境（在项目目录下执行）：

```bash
uv sync
```

激活虚拟环境：

```bash
# Windows (PowerShell/CMD)
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

验证安装：

```bash
python -m wechat_fetcher --help
```

如果看到命令列表（`fetch`、`start-proxy`、`status`），说明安装成功。

本项目提供**两种操作方式**，推荐使用方式A（无需手机）。

## 方式A：Windows 桌面微信（推荐）

电脑上已安装微信桌面版的情况下，**无需手机**，全程在同一台电脑上完成。

### A.1 启动代理

打开一个新的终端窗口，在项目目录下运行：

```bash
python -m wechat_fetcher start-proxy
```

输出示例：

```
WeChat Article Fetcher Proxy
============================================================
Proxy listening on 0.0.0.0:8080

Instructions:
1. Configure your phone's Wi-Fi proxy:
...
```

> 代理会持续运行，**保持此窗口不要关闭**，后续操作在另一个终端窗口中进行。

### A.2 设置 Windows 系统代理

1. 按 `Win + I` 打开 **设置**
2. 点击左侧 **网络和 Internet**
3. 点击右侧 **代理**
4. 在 **手动设置代理** 区域，点击 **使用代理服务器** 旁边的 **设置**（或"编辑"）
5. 将开关设为 **开**，填写：
   - **代理 IP 地址**：`127.0.0.1`
   - **端口**：`8080`
6. 勾选 **"请勿将代理服务器用于本地(Intranet)地址"**（可选，但建议勾选）
7. 点击 **保存**

> **注意**：Windows 系统代理设置后，Edge/Chrome 等浏览器流量也会经过代理。操作完成请记得**关闭代理设置**。

### A.3 安装 mitmproxy CA 证书

**确保代理仍在运行**，然后：

1. 打开浏览器，访问：`http://mitm.it`
2. 点击 **Windows** 图标下载证书文件（`mitmproxy-ca-cert.p12` 或 `.cer`）
3. 双击下载的证书文件，打开 **证书导入向导**
4. 存储位置选择 **当前用户** → 下一步
5. 选择 **"将所有证书放入下列存储"** → 点击 **浏览**
6. 选择 **"受信任的根证书颁发机构"** → 确定 → 下一步
7. 点击 **完成**，在弹出的安全警告中点击 **是**

> **验证证书安装**：按 `Win + R`，输入 `certmgr.msc`，展开 **受信任的根证书颁发机构** → **证书**，找到 `mitmproxy` 即表示安装成功。

### A.4 捕获公众号参数

1. **完全关闭并重新打开**微信桌面版（重要：必须重启微信才能应用新的代理设置）
2. 登录微信
3. 在左侧联系人列表中找到目标公众号，双击打开聊天窗口
4. 点击聊天窗口右上角的 **"小人"头像图标**，进入公众号主页
5. 点击 **"全部消息"**（或向下滚动历史文章列表）
6. **向下滚动鼠标滚轮**，多加载几页文章（滚动到底部让微信自动加载更多）

滚动加载后，代理终端窗口会输出：

```
Captured params for __biz=MzA3MDU1NTIwNg==
```

这表示参数已成功捕获。此时可以按 `Ctrl+C` 停止代理。

> **为什么会捕获不到？** 请参考 [常见问题排查](#常见问题排查)。

### A.5 恢复系统代理

操作完成后，**务必关闭系统代理**：

1. 按 `Win + I` → **网络和 Internet** → **代理**
2. 将 **使用代理服务器** 开关设为 **关**

## 方式B：手机微信

如果你无法在电脑上使用桌面微信，或希望用手机操作，请按照以下步骤。

### B.1 启动代理

```bash
python -m wechat_fetcher start-proxy --show-ip
```

输出示例：

```
WeChat Article Fetcher Proxy
============================================================
Proxy listening on 0.0.0.0:8080

Your local IP: 192.168.1.100
...
```

> **记下显示的 IP 地址**（如 `192.168.1.100`），手机配置时会用到。

### B.2 手机设置 Wi-Fi 代理

#### iOS（iPhone）

1. 打开 **设置** → **无线局域网（Wi-Fi）**
2. 点击当前连接的 Wi-Fi 名称旁边的 ⓘ 图标
3. 滚动到底部，找到 **HTTP 代理**，选择 **手动**
4. 填写：
   - **服务器**：步骤 B.1 中显示的 IP 地址
   - **端口**：`8080`
5. 点击右上角 **存储**

#### Android

1. 打开 **设置** → **WLAN**
2. 长按当前连接的 Wi-Fi，选择 **修改网络**（或点右侧箭头进入高级设置）
3. 找到 **代理** 选项，选择 **手动**
4. 填写：
   - **服务器主机名**：步骤 B.1 中显示的 IP 地址
   - **服务器端口**：`8080`
5. 点击 **保存**

### B.3 安装 CA 证书

**代理必须保持运行状态**，然后在手机浏览器中操作：

1. 打开手机浏览器，访问：`http://mitm.it`
2. 页面会显示证书下载选项

#### iOS 安装步骤

1. 点击 **Apple** 图标下载证书
2. 弹出提示 → 点击 **允许**
3. 打开 **设置** → **通用** → **VPN 与设备管理**
4. 点击下载的配置描述文件 → **安装**
5. 安装完成后，进入 **设置** → **通用** → **关于本机** → **证书信任设置**
6. 找到 `mitmproxy` 证书，**打开信任开关**

#### Android 安装步骤

1. 点击 **Android** 图标下载证书
2. 打开 **设置** → **安全** → **加密与凭据** → **安装证书**（或 **从存储设备安装**）
3. 选择下载的证书文件
4. 命名为 `mitmproxy`，点击确定

> 不同 Android 机型路径略有差异，一般搜索"证书"即可找到入口。

### B.4 捕获公众号参数

1. 确保代理正在运行，手机代理和证书已配置
2. 打开手机微信
3. 找到你想要抓取的目标公众号
4. 进入公众号，点击 **"全部消息"** 或 **"查看历史文章"**
5. **向下滑动文章列表**，多加载几页文章

滑动后，电脑终端上代理窗口会输出：

```
Captured params for __biz=MzA3MDU1NTIwNg==
```

这表示参数已成功捕获。你可以按 `Ctrl+C` 停止代理。

### B.5 恢复手机网络

操作完成后：

1. **移除手机代理**：Wi-Fi 设置 → 代理改回"关闭"或"无"
2. **建议卸载证书**

## 拉取文章URL

参数捕获完成后，在项目目录下执行以下命令。

### 查看已捕获的公众号

```bash
python -m wechat_fetcher status
```

输出示例：

```
Stored accounts (~/.wechat_fetcher/params.json):

  MzA3MDU1NTIwNg==
    Extracted: 2026-05-16T10:30:00+00:00 (120s ago) fresh
```

### 抓取最近30天的文章

```bash
python -m wechat_fetcher fetch --biz MzA3MDU1NTIwNg== --days 30
```

输出示例：

```
Params for MzA3MDU1NTIwNg==
  Key extracted: 2026-05-16T10:30:00+00:00 (120s ago) (fresh)

Fetching articles from last 30 days...

Found 15 articles.

1. Python 3.13 新特性解读
   https://mp.weixin.qq.com/s?__biz=MzA3MDU1NTIwNg==&mid=2651234567&idx=1&sn=abc123...
   2026-05-16

2. 如何提高代码质量
   https://mp.weixin.qq.com/s?__biz=MzA3MDU1NTIwNg==&mid=2651234566&idx=1&sn=def456...
   2026-05-14

...
```

### 导出到文件

```bash
python -m wechat_fetcher fetch --biz MzA3MDU1NTIwNg== --days 30 --output articles.json
```

导出为 JSON 格式，每条记录包含 `title`、`url`、`date` 字段。

### 自定义时间范围

```bash
# 抓取最近7天
python -m wechat_fetcher fetch --biz MzA3MDU1NTIwNg== --days 7

# 抓取最近90天
python -m wechat_fetcher fetch --biz MzA3MDU1NTIwNg== --days 90
```

## 常见问题排查

### A 类问题：Windows 桌面微信

#### A1：设置系统代理后浏览器无法上网

- 确认代理终端窗口仍在运行（`mitmdump` 进程存活）
- 如果不需要浏览器走代理，勾选"请勿将代理服务器用于本地(Intranet)地址"可减少影响
- 操作完成后及时关闭系统代理

#### A2：微信桌面版无法登录或网络异常

- 微信桌面版在某些网络环境下可能会检测代理，尝试以下方法：
  1. 先关闭代理，正常登录微信
  2. 登录成功后，再开启系统代理
  3. 完全关闭微信（任务栏右键退出），重新打开微信
- 如果仍不行，检查 Windows 防火墙是否拦截了 `mitmdump`

#### A3：滑动文章列表但没有"Captured params"输出

- 确认微信**完全退出后重新打开**（任务栏右键 → 退出，而非仅关闭窗口）
- 确认系统代理设置正确（`127.0.0.1:8080`）
- 确认 CA 证书已安装到 **"受信任的根证书颁发机构"**
- 进入公众号主页后需点击 **"全部消息"** 才能看到历史文章列表
- 确保是微信公众号文章（订阅号/服务号），而非视频号或小程序
- 尝试打开公众号中的**任意一篇文章**，然后再回到文章列表滚动加载

### B 类问题：手机微信

#### B1：代理启动后手机无法上网

- 确认电脑防火墙允许 8080 端口入站连接
- 确认手机和电脑在同一 Wi-Fi 下
- 确认手机代理 IP 填写正确（是**电脑的 IP**，不是 `127.0.0.1`）

#### B2：mitm.it 页面打不开

- 确认代理还在运行（终端没有退出）
- 确认手机代理配置正确
- 尝试用 `http://mitm.it` 而不是 `https`
- 检查电脑防火墙是否拦截了来自手机的请求

#### B3：滑动文章列表但没有"Captured params"输出

- 确认证书已安装且**信任开关已打开**（iOS 最关键的一步）
- 尝试**完全关闭微信后重新打开**
- 滑动时尽量**慢一点**，让每页文章加载完成
- 参数 `key` 只在加载文章列表时出现，打开单篇文章不会触发。请确保进入公众号主页并滑动"全部消息"列表

### C 类问题：通用

#### C1：拉取时报错 "Please re-run start-proxy"

- 表示 `key` 已过期（通常有效期为 1-2 小时）
- 解决方法：重新启动代理，在微信中再次打开该公众号并滑动文章列表，然后重新执行 `fetch`

#### C2：抓取过程中提示 "Rate limited"

- 请求频率触发了微信的反爬机制
- 工具会自动等待退避，耐心等待即可，无需手动干预

#### C3：输出结果中有多篇同一天的文章，URL不同

- 这是正常的。微信公众号支持"多图文推送"（一次推送最多 8 篇文章），这些文章共享同一个发布时间，但每篇有独立的 URL。

#### C4：抓取完成后记得做什么？

- **关闭代理**（终端按 `Ctrl+C`）
- **关闭系统代理**（Win+I → 网络和Internet → 代理 → 关闭）或**移除手机代理**
- **建议卸载证书**（特别是在不信任的网络中使用过代理后）

## 安全提醒

1. **`params.json` 包含敏感凭证**：该文件存储了你微信账号的接口鉴权参数（`uin` 和 `key`）。拥有这些参数的人可以在有效期内拉取你可见的公众号文章。请勿分享或上传此文件（已加入 `.gitignore`）。

2. **CA 证书具有高权限**：mitmproxy 的 CA 证书可以解密你设备上**所有** HTTPS 流量。使用完毕后，强烈建议卸载该证书：
   - **Windows**：按 `Win+R` 输入 `certmgr.msc`，展开"受信任的根证书颁发机构"→"证书"，找到 `mitmproxy`，右键 → 删除
   - **iOS**：设置 → 通用 → VPN 与设备管理 → 删除描述文件
   - **Android**：设置 → 安全 → 加密与凭据 → 用户凭据 → 删除 mitmproxy

3. **信任网络环境**：本工具设计用于个人数据归档。请不要在不可信的网络环境（如公共 Wi-Fi）中使用。

4. **合规使用**：本工具利用你自身微信账号的合法权限拉取公开文章信息。请合理使用，遵守微信平台的使用条款。

5. **代理仅转发流量**：本工具不会将你提取的凭证发送给除 `mp.weixin.qq.com`（微信官方服务器）之外的任何第三方。
