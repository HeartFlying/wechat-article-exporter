# 当前项目结构分析

## 项目概述

**项目名称**: wechat-fetcher（微信公众号文章URL抓取工具）

**功能定位**: 通过 MITM（中间人代理）拦截微信 HTTPS 流量，提取公众号鉴权参数，进而批量拉取公众号历史文章 URL。

## 核心模块说明

### 1. `proxy_addon.py` — MITM 代理插件
- **职责**: 作为 mitmproxy addon，拦截流经代理的 HTTPS 请求
- **关键逻辑**:
  - 识别目标 `mp.weixin.qq.com` 的请求
  - 从 URL query、Referer header、Cookie header 中提取 `__biz`、`uin`、`key`、`pass_ticket`、`appmsg_token`
  - 调用 `ParamStore.save()` 持久化参数到本地 JSON
- **依赖**: mitmproxy SDK、wechat_fetcher.storage.ParamStore

### 2. `wechat_fetcher/cli.py` — CLI 入口
- **`start-proxy`**: 启动 mitmdump，加载 `proxy_addon.py`，支持桌面模式和手机模式
- **`fetch`**: 使用已存储的参数从微信 API 拉取文章列表，支持按天数过滤、JSON 导出
- **`status`**: 查看已捕获的公众号参数及其新鲜度

### 3. `wechat_fetcher/fetcher.py` — 文章拉取器
- **`WeChatArticleFetcher`** 类:
  - 调用微信 `profile_ext?action=getmsg` 接口分页拉取
  - 内置重试机制（网络异常、频率限制、key 过期）
  - 频率控制（随机延迟 3-6 秒，反爬退避 60 秒起）
  - 支持多图文推送解析（主文章 + 副文章）
- **`KeyExpiredError`**: key 过期异常

### 4. `wechat_fetcher/storage.py` — 参数存储
- **`ParamStore`** 类:
  - 默认路径 `~/.wechat_fetcher/params.json`
  - 原子写入（tempfile + os.replace）
  - 支持 save / load / list / delete 操作

## 已知特点

1. **双平台支持**: 桌面微信（推荐）和手机微信两种代理模式
2. **参数时效性**: `key` 参数有效期约 1-2 小时，过期需重新捕获
3. **反爬策略**: 随机延迟 + 退避重试 + 多页限制（MAX_PAGES=100）
4. **多图文支持**: 一次推送最多 8 篇独立文章均可识别
5. **安全设计**: 参数文件已加入 `.gitignore`，CA 证书使用后可卸载
