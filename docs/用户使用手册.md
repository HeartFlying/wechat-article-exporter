## 简介

WeChat Fetcher 是一个微信公众号文章抓取工具，通过 MITM 代理拦截微信客户端 HTTPS 流量提取认证参数，进而调用微信文章列表 API 抓取文章 URL，并支持一键下载为 Markdown。

### 主要特性

- **智能参数管理**：自动检测参数过期，支持自动/半自动刷新
- **UI 自动化**：自动操作微信桌面版获取新参数
- **多种使用模式**：手动、半自动、全自动，适应不同场景
- **文章去重**：自动跳过已下载文章，避免重复
- **按公众号组织**：所有数据按公众号名称分子目录存储

## 安装与配置

### 环境要求

- Python 3.9+
- Windows 操作系统（UI 自动化功能）
- 微信桌面版（已登录）

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/HeartFlying/wechat-article-exporter.git
cd wechat-article-exporter

# 2. 创建虚拟环境（推荐）
python -m venv venv
venv\Scripts\activate

# 3. 安装基础依赖
pip install -e .

# 4. 安装 UI 自动化依赖（可选，用于自动刷新功能）
pip install uiautomation psutil

# 5. 安装系统通知依赖（可选）
pip install win10toast
```

### 验证安装

```bash
wechat-fetcher --help
```

## 快速开始

### 首次使用流程

```bash
# 步骤 1: 启动代理获取初始参数
wechat-fetcher start-proxy

# 步骤 2: 操作微信获取参数
# - 保持代理运行
# - 打开微信桌面版
# - 进入目标公众号
# - 点击"全部消息"浏览历史文章
# - 观察代理窗口出现 "Captured params" 日志

# 步骤 3: 停止代理（按 Ctrl+C）

# 步骤 4: 查看已获取的参数
wechat-fetcher status

# 步骤 5: 执行抓取
wechat-fetcher run --all --days 7
```

## 核心功能

### 1. 参数健康检查

工具会自动跟踪参数的过期时间，提供三种状态：

- **✓ 有效**：参数正常，可以使用
- **! 即将过期**：参数将在 30 分钟内过期，建议刷新
- **✗ 已过期**：参数已失效，必须刷新

```bash
# 查看所有账号的健康状态
wechat-fetcher status --health

# 查看需要关注的账号
wechat-fetcher health --notify
```

### 2. 自动刷新参数

当参数过期时，可以使用 UI 自动化自动刷新：

```bash
# 刷新单个账号
wechat-fetcher refresh --account "公众号名称" --auto

# 刷新所有过期账号
wechat-fetcher refresh --auto

# 启动自动刷新守护进程（每 5 分钟检查一次）
wechat-fetcher refresh --daemon --auto --interval 5
```

### 3. 文章抓取与下载

```bash
# 抓取最近 7 天的文章
wechat-fetcher run --all --days 7

# 抓取指定日期范围
wechat-fetcher run --all --from 2024-01-01 --to 2024-01-31

# 只获取 URL 列表（不下载）
wechat-fetcher fetch --biz <biz值> --days 30

# 下载文章为 Markdown
wechat-fetcher download --input articles.json
```

## 使用模式

### 模式一：全自动模式（推荐用于服务器）

适合长期无人值守运行。

**终端 1 - 启动自动刷新守护进程：**
```bash
wechat-fetcher refresh --daemon --auto --interval 10
```

**终端 2 - 定时执行抓取（使用 Windows 任务计划程序）：**
```bash
wechat-fetcher run --all --days 1
```

**流程说明：**
1. 守护进程每 10 分钟检查参数状态
2. 发现过期时自动操作微信获取新参数
3. 定时任务使用有效参数抓取文章

### 模式二：半自动模式（推荐用于个人电脑）

适合日常使用，保持控制的同时减少手动操作。

```bash
# 1. 检查参数状态
wechat-fetcher status --health

# 2. 如有过期，执行刷新
wechat-fetcher refresh --auto

# 3. 执行抓取
wechat-fetcher run --all --days 3
```

### 模式三：手动模式

适合初次配置或调试。

```bash
# 1. 启动代理
wechat-fetcher start-proxy

# 2. 手动操作微信获取参数
# 3. 停止代理

# 4. 查看状态并执行抓取
wechat-fetcher status
wechat-fetcher run --all --days 7
```

### 模式四：定时任务模式

使用 Windows 任务计划程序定期执行。

**创建批处理文件 `daily_fetch.bat`：**
```batch
@echo off
echo [%date% %time%] 开始执行...

wechat-fetcher health --notify
wechat-fetcher refresh --auto
wechat-fetcher run --all --days 1

echo [%date% %time%] 执行完成
```

**配置任务计划程序：**
1. 打开"任务计划程序"
2. 创建基本任务
3. 设置触发器（每天凌晨 2 点）
4. 设置操作为启动程序 `daily_fetch.bat`

## 命令参考

### 核心命令

| 命令 | 功能 | 常用示例 |
|------|------|----------|
| `start-proxy` | 启动 MITM 代理 | `wechat-fetcher start-proxy --show-ip` |
| `status` | 查看账号状态 | `wechat-fetcher status --health` |
| `health` | 健康检查 | `wechat-fetcher health --notify` |
| `refresh` | 自动刷新参数 | `wechat-fetcher refresh --auto` |
| `run` | 一键抓取 | `wechat-fetcher run --all --days 7` |
| `fetch` | 获取 URL 列表 | `wechat-fetcher fetch --biz XXX --days 30` |
| `download` | 下载文章 | `wechat-fetcher download --input articles.json` |
| `watch` | 监控参数状态 | `wechat-fetcher watch --once` |
| `clear` | 清除数据 | `wechat-fetcher clear --name XXX` |
| `info` | 显示信息 | `wechat-fetcher info` |

### 命令详细说明

#### start-proxy

启动 MITM 代理拦截微信请求。

```bash
wechat-fetcher start-proxy [选项]

选项:
  --port INTEGER       代理端口 (默认: 8080)
  --listen-host TEXT   监听地址 (默认: 0.0.0.0)
  --show-ip           显示本地 IP 用于手机代理配置
```

#### status

查看已存储的账号参数和去重索引状态。

```bash
wechat-fetcher status [选项]

选项:
  -v, --verbose    显示完整的 key/uin 值
  -h, --health     显示详细的健康检查结果
```

#### health

检查账号参数的健康状态和有效期。

```bash
wechat-fetcher health [选项]

选项:
  --biz TEXT    检查指定账号的健康状态
  --notify      仅显示需要关注的账号（已过期或即将过期）
```

#### refresh

自动刷新过期参数。

```bash
wechat-fetcher refresh [选项]

选项:
  -a, --account TEXT    指定要刷新的公众号名称
  --auto                自动执行 UI 自动化（无需确认）
  -d, --daemon          以守护进程模式运行
  -i, --interval INTEGER  检查间隔（分钟，默认: 5）
  -s, --strategy [auto|semi|notify]  刷新策略
```

**策略说明：**
- `auto`: 完全自动，检测到过期立即执行
- `semi`: 半自动，需要 `--auto` 参数确认（默认）
- `notify`: 仅通知，不执行操作

#### run

一键工作流：抓取 → 去重 → 下载。

```bash
wechat-fetcher run [选项]

选项:
  --biz TEXT          目标账号 __biz (可多次指定)
  --all               处理所有已存储账号
  --days INTEGER      抓取最近 N 天的文章
  --from TEXT         开始日期 (YYYY-MM-DD)
  --to TEXT           结束日期 (YYYY-MM-DD)
  -o, --output-dir TEXT  输出目录
  --cookie TEXT       微信认证 Cookie
  --wait-proxy        等待代理捕获参数后再开始
```

#### watch

启动定时健康检查守护进程。

```bash
wechat-fetcher watch [选项]

选项:
  -i, --interval INTEGER  检查间隔（分钟，默认: 30）
  -n, --notify            启用系统通知
  --once                  只执行一次检查
```

## 故障排查

### 常见问题

#### 1. 参数过期错误

**现象：**
```
错误: Key 已过期，请运行 start-proxy 刷新参数后重试
```

**解决：**
```bash
# 方式 1: 自动刷新
wechat-fetcher refresh --auto

# 方式 2: 手动刷新
wechat-fetcher start-proxy
# 操作微信...
```

#### 2. UI 自动化失败

**现象：**
```
UI 自动化失败: 无法找到微信窗口
```

**解决：**
1. 确保微信已登录
2. 确保微信窗口没有被最小化
3. 以管理员身份运行命令提示符
4. 检查依赖是否安装：`pip install uiautomation psutil`

#### 3. 找不到 mitmdump

**现象：**
```
错误: 未找到 mitmdump
```

**解决：**
```bash
pip install mitmproxy
```

#### 4. 抓取失败

**现象：**
文章列表为空或下载失败

**解决：**
1. 检查参数健康状态：`wechat-fetcher health`
2. 刷新过期参数：`wechat-fetcher refresh --auto`
3. 减少日期范围重试：`--days 3`
4. 检查网络连接

#### 5. 权限错误

**现象：**
```
PermissionError: [Errno 13] Permission denied
```

**解决：**
1. 以管理员身份运行命令提示符
2. 检查数据目录权限
3. 关闭杀毒软件重试

### 调试技巧

```bash
# 查看详细日志
wechat-fetcher -v status

# 检查数据目录
wechat-fetcher info

# 验证参数有效性
wechat-fetcher health --biz <biz值>

# 测试单个账号
wechat-fetcher fetch --biz <biz值> --days 1
```

## 最佳实践

### 1. 首次配置

1. 确保微信已登录且保持在线
2. 使用 `start-proxy` 获取初始参数
3. 验证参数获取成功：`status --health`
4. 测试小规模抓取：`run --all --days 1`

### 2. 日常使用

1. 定期执行 `health --notify` 检查参数状态
2. 参数即将过期时及时刷新
3. 使用 `run --all` 批量抓取
4. 定期清理旧数据：`clear --name XXX`

### 3. 长期运行

1. 使用 `refresh --daemon` 保持参数更新
2. 配置定时任务定期抓取
3. 监控日志文件
4. 定期备份数据目录

### 4. 数据管理

数据目录结构：
```
~/.wechat-fetcher/data/
├── accounts/
│   ├── 公众号名称A/
│   │   ├── params.json       # 认证参数
│   │   ├── dedup_index.json  # 去重索引
│   │   └── articles/         # 下载的文章
│   │       └── ...
│   └── 公众号名称B/
│       └── ...
```

**备份建议：**
```bash
# 备份数据
xcopy %USERPROFILE%\.wechat-fetcher\data D:\backup\wechat-data /E /I

# 恢复数据
xcopy D:\backup\wechat-data %USERPROFILE%\.wechat-fetcher\data /E /I
```

### 5. 安全建议

1. 不要分享 `~/.wechat-fetcher/data/accounts/` 目录
2. 包含微信账号的接口鉴权参数
3. 使用完毕后建议卸载 mitmproxy CA 证书
4. 合理使用，避免频繁请求
